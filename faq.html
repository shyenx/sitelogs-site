Frequently asked questions

Everything people ask before (and after) installing. If your question isn't here, drop us an email or open an issue.

Privacy & data

Will my browsing data ever leave my device?+

No. SiteLogs has no server. Visit data lives in your browser's IndexedDB on this device only. The only outbound calls are Google Fonts CSS (will self-host) and our license-key activation flow when you pay. See the full Privacy Policy.

Does my visit data sync across devices?+

No — and on purpose. Your settings (rules, tags, pinned sites, budgets) sync via Chrome Sync if you have it on. Visit records stay on each device, so the maintainer never has access to them.

Can I export or wipe my data?+

Yes — Settings → Data → Export JSON dumps everything. Wipe all data clears IndexedDB and chrome.storage. Uninstalling the extension also removes everything stored in your browser.

What does SiteLogs not record?+

Page content, form input, passwords, cookies, network requests, anything from chrome:// / chrome-extension:// / file:// URLs, anything from incognito windows, and anything from domains on your ignore list.

Can I import my existing Chrome history?+

Yes. On first install, a modal asks if you'd like to pre-populate the dashboard from your Chrome history (30 days / 90 days / 1 year / All). It's opt-in — pick "Skip" if you'd rather start fresh. You can also re-run the import any time from Settings → Data.

Activation & devices

Can I use one license on multiple devices?+

Yes — up to 5 devices on a single license key. Free up a slot from Settings → License → Deactivate this device.

I lost my license key. How do I recover it?+

Search your inbox for "SiteLogs license" — the original purchase confirmation email contains it. If you can't find it, email us with the address you used to buy.

I reinstalled and lost a device slot. What do I do?+

Reinstalling generates a new device ID, so it counts as a new slot. If you've used all 5 slots, email us and we'll reset slots on your license key — usually within a business day.

Browser & compatibility

Which browsers are supported?+

Chrome 120+ today. Edge / Brave / Vivaldi (Chromium-based) typically work but aren't officially supported yet. Firefox / Safari ports are on the roadmap, no ETA.

Does it work in incognito?+

No — Chrome blocks extensions in incognito by default, and we don't override that. It's intentional: if you're in incognito, you presumably don't want anything (including SiteLogs) tracking the session.

Will it slow down my browser?+

No measurable slowdown. The extension runs a tiny content script (a few hundred bytes) that posts visibility events to a background service worker. The dashboard only renders when you open a new tab. Total disk: < 200 KB.

Features

What's "active time" exactly?+

Time the tab was focused AND you weren't idle. Chrome's native idle API marks you idle after 60s without keyboard or mouse input — SiteLogs pauses the timer for that period. So if you open a tab and walk away for 10 min, those 10 min don't count.

How does category classification work?+

A 200+ entry built-in dictionary maps domains to one of 13 categories (AI, Tech, Social, Video, News, Shopping, Docs, Tools, Work, Finance, Research, Leisure, Other). You can override any domain with a custom rule, or use regex rules for URL patterns. First match wins.

Can I pin sites to the new tab?+

Yes — hover any "hot site" tile on the Overview, click the pin icon. Pinned tiles stay first, even if usage drops. Settings sync via Chrome Sync if you have it on, so pins follow your profile across machines.

What's a "Session"?+

A named snapshot of all tabs in a window. Use it to switch contexts (work / research / personal). Save: Tabs page → "Save as session". Restore: Sessions page → "Restore in new window". Sessions are stored locally and don't sync.

Does SiteLogs support Chinese?+

Yes — full Chinese localization for every label, message, category name, and FAQ. Switch in Settings.

Are there keyboard shortcuts?+

Yes. Press ? in the dashboard to see all of them — Overview / Tabs / History / Sessions / Tags / Settings each have a leader-key combo.
